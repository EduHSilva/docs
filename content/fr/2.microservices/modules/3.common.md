# Objetivo

O módulo **Comum** (ou *Common*) é uma biblioteca compartilhada que visa **centralizar DTOs (Data Transfer Objects), contratos e padrões** de arquitetura utilizados por todos os microsserviços do sistema. Sua função é garantir a consistência, padronização e interoperabilidade entre os diferentes componentes da aplicação.

## Como Adicionar em Novos Serviços

Para incluir o módulo Comum como dependência em um novo microsserviço (exemplo em um projeto Maven):

```xml
<dependency>
    <groupId>com.edu.silva</groupId>
	<artifactId>common</artifactId>
	<version>0.0.1-SNAPSHOT</version>
</dependency>
```

## Padrões de Resposta da API

Todas as APIs do sistema devem seguir um padrão de resposta unificado, facilitando a integração com clientes (front-end) e outros microsserviços.

### Resposta para Item Simples

Utilizado para operações que retornam um único recurso (ex: `findOne`, `save`, `update`).

```json
{
    "data": {},
    "links": null,
    "message": ""
}
```

| Campo | Tipo | Descrição |
| :--- | :--- | :--- |
| `data` | Objeto | O recurso principal solicitado ou o resultado da operação. |
| `links` | Objeto/Null | Links HATEOAS opcionais para ações relacionadas. |
| `message` | String | Mensagem de status da operação (ex: "Usuário salvo com sucesso"). |

### Resposta para Item Paginado

Utilizado para operações de listagem de recursos (ex: `getAll`).

```json
{
  "data": {
    "items": [{}],
    "page": 0,
    "size": 0,
    "total": 0
  },
  "links": null,
  "message": ""
}
```

| Campo | Tipo | Descrição |
| :--- | :--- | :--- |
| `data.items` | Array | A lista de recursos retornados na página atual. |
| `data.page` | Inteiro | O número da página atual (base 0). |
| `data.size` | Inteiro | O tamanho máximo da página. |
| `data.total` | Inteiro | O número total de recursos disponíveis. |
| `links` | Objeto/Null | Links HATEOAS opcionais (ex: `self`, `next`, `prev`). |
| `message` | String | Mensagem de status da operação (ex: "Clientes encontrados com sucesso"). |

## Observações

- **Consistência:** O uso deste módulo garante que todas as APIs do sistema sigam o mesmo padrão de resposta, o que é crucial para a experiência do desenvolvedor (DX).
- **Integração:** O padrão de resposta facilita a integração com o front-end e outros sistemas consumidores, pois o formato dos dados é previsível.
- **Evolução:** O padrão de encapsulamento (`data`, `links`, `message`) permite a evolução futura da API (ex: adição de metadados ou HATEOAS) sem quebrar o contrato básico com os clientes existentes.
