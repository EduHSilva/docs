## Objetivo
Centralizar **DTOs, contratos e padrões** utilizados entre os microservices.

## Como adicionar em novos serviços
```xml
<dependency>
    <groupId>com.edu.silva</groupId>
	<artifactId>common</artifactId>
	<version>0.0.1-SNAPSHOT</version>
</dependency>
```

## Respostas padrão
### Item simples
```json
{
    "data": {},
    "links": null,
    "message": ""
}
```

### Item paginado

```json
{
  "data": {
    "items": [{}],
    "page": 0,
    "size": 0,
    "total": 0
  },
  "links": null,
  "message": ""
}
```
### Observações
- Todas as APIs seguem esse padrão de resposta
- Facilita integração com frontend e outros sistemas
- Permite evolução sem quebra de contrato
