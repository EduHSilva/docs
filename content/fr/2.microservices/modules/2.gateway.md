## Objetivo
**Centralizar e padronizar** as chamadas entre clientes e microservices.
O gateway atua como:
- Ponto único de entrada
- Validador de autenticação
- Roteador de requisições

## Rotas
Todo microservice registrado no service discover fica acessível através do padrão:```
```
http://localhost:8080/{nome-do-servico}
```
### Rota pública de teste
Cada microservice expõe uma rota pública /ping para validação de disponibilidade.
```http request
GET http://localhost:8080/users/ping
```
Resposta:
```json
{
  "data": null,
  "links": null,
  "message": "pong"
}

```

## Rotas autentificadas
Para acessar rotas protegidas é necessário enviar o header:
```
Authorization: Bearer <token>
```
O gateway:
- Valida o token JWT
- Extrai o usuário autenticado
- Repassa a requisição ao microservice com o header:
```
X-User-ID
```
