## Objetivo

Responsável por **registrar e disponibilizar** os microservices para o gateway.

O service discover atua como um catálogo dinâmico, permitindo que novos serviços sejam adicionados ou removidos sem necessidade de reconfiguração manual do gateway.

## Funcionamento
- Cada microservice se registra automaticamente ao iniciar
- O gateway consulta o service discover para resolver o destino das requisições
- Permite escalabilidade horizontal (múltiplas instâncias do mesmo serviço)

## Interface
Disponível em:
```
http://localhost:8761/
```

Na home page é possível:
- Visualizar serviços registrados
- Ver status (UP / DOWN)
- Ver quantidade de instâncias por serviço
