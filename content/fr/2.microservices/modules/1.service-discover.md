# Objetivo

O módulo de **Service Discovery** é um componente fundamental em arquiteturas de microsserviços. Seu principal objetivo é **registrar e disponibilizar** de forma dinâmica a localização de todos os microsserviços para o Gateway de API e outros serviços consumidores.

Ele atua como um **catálogo dinâmico**, permitindo que novos serviços sejam adicionados ou removidos da arquitetura sem a necessidade de reconfiguração manual do Gateway ou de outros serviços.

## Funcionamento

O Service Discovery opera com base em um mecanismo de auto-registro e consulta:

- **Auto-Registro:** Cada microsserviço se registra automaticamente no Service Discovery ao ser inicializado, informando seu endereço e porta.
- **Consulta pelo Gateway:** O Gateway de API consulta o Service Discovery para resolver o endereço de destino de cada requisição, utilizando o nome lógico do serviço.
- **Escalabilidade Horizontal:** Ao permitir o registro de múltiplas instâncias do mesmo serviço, o Service Discovery facilita a implementação de balanceamento de carga e escalabilidade horizontal.

## Interface de Gerenciamento

A interface de gerenciamento do Service Discovery está disponível no seguinte endereço:

```
http://localhost:8761 
```

Através desta interface, é possível monitorar o estado atual dos microsserviços registrados:

- **Visualizar Serviços Registrados:** Lista todos os microsserviços que estão ativos e registrados.
- **Ver Status (UP / DOWN):** Exibe o status de saúde de cada instância de serviço.
- **Ver Quantidade de Instâncias:** Mostra o número de instâncias em execução para cada microsserviço.

## Observações

- **Desacoplamento:** O Service Discovery é essencial para o desacoplamento, pois os serviços não precisam saber a localização física uns dos outros, apenas seus nomes lógicos.
- **Resiliência:** Ao monitorar o status dos serviços, ele permite que o Gateway e outros clientes evitem enviar tráfego para instâncias que falharam.
